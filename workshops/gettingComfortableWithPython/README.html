Formatted(f) strings

Also known as string interpolation

Works in python 3.6 and above

Normal string

def get_all_results(table_name):
     return run_query(
          "SELECT * from " + table_name + ";"
     )

Using f string

def get_all_results(table_name):
     return run_query(
          f"SELECT * from {table_name};"
     )

Raw(r) strings

Normal string

with open("C:\\Some\\file\\name.txt") as f:
    print(f.read(100))

Using r string

with open(r"C:\Some\file\name.txt") as f:
    print(f.read(100))

How are they different

print(r'Hello\tWorld' != 'Hello\tWorld')

True

How are they same

print(r'Hello\tWorld' == 'Hello\\tWorld')

True

Exception(Error) handling

Sample Code

student = {
    "name": "Ram"
}
print(student["mother"])

Find the error.

Error

Traceback (most recent call last):
File "<stdin>", line 4, in <module>
KeyError: 'mother'

Here type of the error is KeyError. All errors have this information at the end of error output.

Catching errors

student = {
    "name": "Ram"
}
try:
    print(student["mother"])
except KeyError as error:
    print(f"Key not found {error}")

Key not found 'mother'

Catching multiple errors?

student = {
    "name": "Ram",
    "roll": 1
}
keys = ["name", "roll", "dob"]
for key in keys:
    print("Output:", student[key] + 1)

Catching multiple errors (Solution)

student = {
    "name": "Ram",
    "roll": 1
}
keys = ["name", "roll", "dob"]
for key in keys:
    try:
        print("Output:", student[key] + 1)
    except KeyError as error:
        print(f"Key not found {error}")
    except TypeError:
        print(f"Can't add 1 to '{key}'")
    except Exception as error:
        # Catches all other errors
        print(f"Key: '{key}', Error: {error}")
